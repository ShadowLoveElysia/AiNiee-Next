### V2.1.0 更新日志 (2026-01-03)

#### ✨ 新功能与优化 (Features & Optimizations)

1.  **配置编辑体验升级**：
    *   在“术语表”、“排除列表”和“API 配置”菜单中新增**“在编辑器中打开”**功能，可直接调用系统默认编辑器修改复杂的 JSON 配置，保存后自动重载。

2.  **智能路径输入**：
    *   优化了所有路径输入，可自动清洗拖拽文件到终端时产生的多余空格和引号。
    *   当输入一个文件夹路径时，系统会智能检测内部是否仅存在一个 `.txt` 或 `.epub` 文件，并询问是否直接处理该文件。

3.  **快速上手向导 (Onboarding Wizard)**：
    *   首次运行程序时，将启动一个设置向导，引导新用户快速完成界面语言、翻译语言、API 平台选择及验证等核心配置。

4.  **高级错误处理与日志过滤**：
    *   当 API 请求连续失败达到可配置的阈值时，任务将**自动暂停**并高亮显示严重错误，等待用户干预。
    *   任务进行中，可按快捷键 `v` 切换**“全部日志”**和**“仅错误日志”**视图，便于排查问题。

5.  **任务中动态调整 (Hot-Swap)**：
    *   **动态调整分割限制**：运行时可通过快捷键 `[` / `]` 动态增减每次请求的行数或 Token 限制。
    *   **跳过当前文件**：新增快捷键 `n`，允许在任务中跳过当前卡住的文件，继续处理后续文件。

6.  **增强的断点续传提示**：
    *   当检测到输出目录已存在时，新增**“归档旧目录并重新开始”**选项，可自动备份旧的结果文件夹，安全地开始新任务。

7.  **稳定性提升**：
    *   为日志系统增加了**熔断机制**，可防止 UI 渲染错误导致的无限递归崩溃。
    *   改进了键盘监听器，在 SSH 或 Docker 等非标准终端环境中会自动禁用快捷键功能并给出提示，避免程序卡死或崩溃。

#### 🐛 Bug 修复 (Bug Fixes)

*   修复了因 `on_skip_file_request` 方法缺失导致的 `AttributeError` 启动崩溃问题。
